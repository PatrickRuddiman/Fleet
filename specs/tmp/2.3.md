# Task 2.3: Implement env_hash computation utility

## WHAT

Create a utility function `computeEnvHash(exec: ExecFn, envFilePath: string): Promise<string | null>` that computes the SHA256 hash of a `.env` file on the remote server. Returns `null` if the file does not exist. The hash is stored at the stack level in `state.json` (not per-service).

The hash is computed on the remote host via:

```
sha256sum <envFilePath>
```

Fleet parses the hex digest from the first whitespace-delimited field of the output and returns `"sha256:" + hexdigest`.

## WHY

V1.2 spec §2.4 — the `env_hash` detects when the `.env` file content has changed between deploys. If it changes, all long-running services in the stack are restarted (even if their image and definition are unchanged) so they pick up the new environment variables. Hashing the whole file avoids the complexity of tracking which variables each service actually consumes. The hash is stored at the stack level rather than per-service because a single `.env` file applies to all services in the stack.

## HOW

1. Add `computeEnvHash(exec: ExecFn, envFilePath: string): Promise<string | null>` to `src/deploy/hashes.ts` (same file as the functions introduced in tasks 2.1 and 2.2).

2. Run `sha256sum <envFilePath>` via `exec()`. If `result.code !== 0`, return `null`. A non-zero exit code means the file does not exist or is not readable — treat this as "no env file for this stack."

3. Parse stdout: split on whitespace and take the first token (the hex digest). Return `"sha256:" + hexdigest`.

4. The `envFilePath` argument is the remote absolute path to the `.env` file — typically `${stackDir}/.env` where `stackDir` comes from the `DeployContext` already threaded through the deploy helpers.

5. Timing: this function is called AFTER the new `.env` file has been uploaded to the remote host (step 10 in the deploy flow, using the atomic `.tmp` + `mv` pattern). This guarantees the hash always reflects the content that was just deployed, not the previous file.

6. Export `computeEnvHash` alongside the other hash utilities from the deploy module's barrel export.

7. Write a unit test in `tests/deploy/hashes.test.ts` (alongside the task 2.1 and 2.2 tests) that mocks `exec` and verifies:
   - Present file: stdout `"abc123def456  /path/to/.env\n"` → returns `"sha256:abc123def456"`
   - Missing file: non-zero exit code → returns `null`
   - Malformed output (empty stdout despite exit code 0) → define and document the expected behavior (e.g., return `null` or throw — pick one and be consistent with how `getImageDigest` handles analogous edge cases)
