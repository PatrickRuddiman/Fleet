# Task 1.1: Extend compose parser to capture full service definition

## WHAT

Extend `ParsedService` in `src/compose/types.ts` to capture all fields needed for definition_hash computation and one-shot detection. Currently `ParsedService` only has `hasImage`, `hasBuild`, and `ports: NormalizedPort[]`. Add the following optional fields:

- `image` (string | undefined)
- `command` (string | string[] | undefined)
- `entrypoint` (string | string[] | undefined)
- `environment` (Record<string,string> | string[] | undefined)
- `volumes` (string[] | undefined)
- `labels` (Record<string,string> | string[] | undefined)
- `user` (string | undefined)
- `working_dir` (string | undefined)
- `healthcheck` (object | undefined)
- `restart` (string | undefined)

Update `src/compose/parser.ts` (`loadComposeFile`) to extract these new fields from the raw YAML object when parsing each service. Follow the existing pattern: read from the raw YAML service object, normalize nulls to undefined.

## WHY

V1.2 spec §2.2 defines a definition_hash covering these specific fields. V1.2 spec §2.5 uses `restart` to detect one-shot services. Neither is possible without the compose parser capturing them. The current `ParsedService` is too minimal to support the V1.2 selective deployment feature — all downstream hash computation and one-shot detection logic depends on having these values available at parse time.

## HOW

1. Open `src/compose/types.ts`. Add the new optional fields listed above to the `ParsedService` interface, each marked with `?`.

2. Open `src/compose/parser.ts`. In `loadComposeFile`, for each service in the raw YAML, extract the new fields from `rawService.image`, `rawService.command`, `rawService.entrypoint`, `rawService.environment`, `rawService.volumes`, `rawService.labels`, `rawService.user`, `rawService.working_dir`, `rawService.healthcheck`, and `rawService.restart`. Assign each to the corresponding new field on the `ParsedService` object being constructed. Convert `null` values to `undefined` to match TypeScript types (e.g., `rawService.image ?? undefined`).

3. Open `src/compose/queries.ts`. Verify that existing query functions such as `findServicesWithoutImageOrBuild` still compile and behave correctly — they check `hasImage` and `hasBuild` which are unchanged. No functional change is required here, but confirm nothing breaks.

4. Update any tests in `tests/` that construct `ParsedService` objects directly as literals. Because the new fields are optional (`?`), existing test literals remain valid TypeScript without modification. However, add at least one test that verifies the parser correctly extracts the new fields from a minimal YAML fixture containing `image`, `restart`, `command`, and `environment`.
