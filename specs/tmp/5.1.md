# 5.1 — Add --force / -f flag to fleet deploy

## WHAT

Add a `--force` / `-f` boolean flag to the `fleet deploy` CLI command. When set, all change detection is bypassed: every service is pulled and redeployed unconditionally. Extend `DeployOptions` to include `force: boolean`. Thread the value through `DeployContext` or pass it to the steps that need it (selective pull in step 10, selective deploy in step 11).

## WHY

V1.2 spec §4 — `--force` is the escape hatch for situations where `state.json` may be out of sync with reality (external Docker changes, debugging). It guarantees a clean full redeploy. Without it, operators have no way to override change detection when they know something changed outside of fleet.

## HOW

1. Open `src/commands/deploy.ts`. Register the new flag: `.option('-f, --force', 'Bypass change detection and redeploy all services')`. Follow the pattern of existing flags (`--skip-pull`, `--dry-run`).
2. Open `src/deploy/types.ts`. Add `force: boolean` to `DeployOptions`.
3. In `src/commands/deploy.ts`, pass `force: options.force ?? false` into the `DeployOptions` object passed to the deploy function.
4. In `src/deploy/deploy.ts`, when `options.force` is true:
   - Skip the classification step (or pass `force=true` through to skip it)
   - Pass `force=true` to `pullSelectiveImages` → pulls all images
   - Pass `force=true` to the selective deploy step → runs `docker compose up -d --remove-orphans`
5. Force mode does NOT skip hash computation or `state.json` updates — hashes are still computed and stored so the next non-forced deploy has accurate state to compare against.
6. Ensure `-f` short form works (commander supports short + long flag on same `.option()` call).
