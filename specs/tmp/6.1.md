# 6.1 — Deploy output formatting for selective / force / nothing-changed modes

## WHAT

Update `printSummary` (or equivalent output function) in `src/deploy/helpers.ts` to display per-service status lines showing what happened to each service during the deploy. Support three output modes:

**Selective mode** (normal): show each service with its reason and outcome:
```
  migrate      one-shot   → running
  api          image changed (abc123 → def456)  → deploying
  docs         env changed  → restarting
  postgres     no changes   → skipped  (last deployed 4 days ago)
```

**Force mode**: show each service marked as forced:
```
⚠ Force mode — all services will be redeployed
  api          → deploying (forced)
```

**Nothing-changed mode** (all skipped): same as selective but every service shows `no changes → skipped`.

Summary line: `2 deployed, 1 restarted, 1 run (one-shot), 2 skipped`

## WHY

V1.2 spec §7 — operators need visibility into what fleet did and why. Without per-service status lines, selective deployment is a black box: operators can't tell if postgres was intentionally skipped or accidentally missed. The summary line enables quick scanning in CI logs.

## HOW

1. Open `src/deploy/helpers.ts`. Refactor or extend `printSummary` to accept `classification: ServiceClassification`, `force: boolean`, and `changeReasons: Map<string, string>` (a map from service name to the human-readable reason for the action, e.g. `"image changed (abc → def)"`).
2. For each service in compose order, print an aligned status line:
   - One-shot (in toDeploy): `<name>   one-shot   → running`
   - toDeploy (with image change): `<name>   image changed (<old_tag> → <new_tag>)  → deploying`
   - toDeploy (not previously running): `<name>   new service   → deploying`
   - toRestart: `<name>   env changed  → restarting`
   - toSkip: `<name>   no changes   → skipped  (last deployed X ago)`
   - forced: `<name>   → deploying (forced)`
3. Print the `⚠ Force mode` banner before the service list when `force=true`.
4. After the service list, print the summary line: `N deployed, M restarted, P run (one-shot), Q skipped`.
5. The "last deployed X ago" time is derived from `ServiceState.deployed_at` in state — format relative time (e.g., "2 hours ago", "4 days ago").
6. Align service names and reasons using the longest name as the column width, following the existing alignment approach in `formatTable` in `ps.ts`.
